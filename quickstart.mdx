This guide walks you through creating a complete refundable payment flow using the x402r SDK. You'll learn how to set up viem clients, create an x402r client, and execute core payment operations.

## Prerequisites

- Node.js 22 or higher installed
- Basic familiarity with TypeScript and Ethereum development
- An Ethereum wallet with test ETH on Base Sepolia (or another supported testnet)

<Info>
  This guide uses Base Sepolia testnet. You can get test ETH from the [Base Sepolia faucet](https://www.coinbase.com/faucets/base-ethereum-goerli-faucet).
</Info>

## Step-by-Step Guide

<Steps>

<Step title="Install Dependencies">

Install the x402r SDK and viem:

<CodeGroup>

```bash npm
npm install @x402r/sdk viem
```

```bash yarn
yarn add @x402r/sdk viem
```

```bash pnpm
pnpm add @x402r/sdk viem
```

</CodeGroup>

</Step>

<Step title="Set Up Viem Clients">

Create a `PublicClient` and `WalletClient` using viem. The x402r SDK uses these clients for all blockchain interactions:

```typescript
import { createPublicClient, createWalletClient, http } from 'viem'
import { baseSepolia } from 'viem/chains'
import { privateKeyToAccount } from 'viem/accounts'

// Create a public client for reading blockchain data
const publicClient = createPublicClient({
  chain: baseSepolia,
  transport: http(),
})

// Create a wallet client for signing transactions
const account = privateKeyToAccount('0x...')  // Your private key
const walletClient = createWalletClient({
  chain: baseSepolia,
  transport: http(),
  account,
})
```

<Warning>
  Never hardcode private keys in production code. Use environment variables or secure key management solutions.
</Warning>

</Step>

<Step title="Create x402r Client">

Initialize the x402r client with your viem clients and operator address:

```typescript
import { createX402r } from '@x402r/sdk'

const x402r = createX402r({
  publicClient,
  walletClient,
  operatorAddress: '0x...', // The operator managing escrow
})
```

The `operatorAddress` is the on-chain address of the x402r operator contract that manages payment escrow and dispute resolution.

<Tip>
  You can also use role-based presets for type-narrowed clients:
  ```typescript
  import { createPayerClient, createMerchantClient } from '@x402r/sdk'
  
  const payer = createPayerClient({ publicClient, walletClient, operatorAddress: '0x...' })
  const merchant = createMerchantClient({ publicClient, walletClient, operatorAddress: '0x...' })
  ```
</Tip>

</Step>

<Step title="Query Operator Configuration">

Fetch the operator's configuration to understand fees and policies:

```typescript
const config = await x402r.operator.getConfig()

console.log('Operator Config:', {
  arbiter: config.arbiter,
  minFeeBps: config.minFeeBps,
  maxFeeBps: config.maxFeeBps,
  defaultFeeBps: config.defaultFeeBps,
})
```

The operator config includes fee boundaries (in basis points) and the arbiter address for dispute resolution.

</Step>

<Step title="Check Payment State">

Query the current state of a payment using a `PaymentInfo` object:

```typescript
import type { PaymentInfo } from '@x402r/sdk'

// PaymentInfo identifies a unique payment
const paymentInfo: PaymentInfo = {
  payer: '0x...', // Payer's address
  receiver: '0x...', // Merchant's address
  token: '0x...', // Token contract address (use zero address for native ETH)
  amount: BigInt('1000000000000000000'), // Amount in wei (1 ETH)
  operator: '0x...', // Same as operatorAddress above
}

const state = await x402r.payment.getState(paymentInfo)

console.log('Payment State:', {
  isAuthorized: state[0],
  authorizedAmount: state[1],
  chargedAmount: state[2],
})
```

</Step>

<Step title="Get Payment Amounts">

Retrieve detailed amount breakdowns including fees and refundable amounts:

```typescript
const amounts = await x402r.payment.getAmounts(paymentInfo)

console.log('Payment Amounts:', {
  authorizedAmount: amounts.authorizedAmount,
  chargedAmount: amounts.chargedAmount,
  receiverAmount: amounts.receiverAmount,
  operatorFee: amounts.operatorFee,
  refundableAmount: amounts.refundableAmount,
})
```

This returns a structured breakdown of:
- **authorizedAmount**: Total authorized in escrow
- **chargedAmount**: Amount captured by the merchant
- **receiverAmount**: Net amount the merchant receives (after fees)
- **operatorFee**: Fee collected by the operator
- **refundableAmount**: Amount available for refund

</Step>

</Steps>

## Complete Working Example

Here's a full example that brings it all together:

```typescript
import { createPublicClient, createWalletClient, http } from 'viem'
import { baseSepolia } from 'viem/chains'
import { privateKeyToAccount } from 'viem/accounts'
import { createX402r } from '@x402r/sdk'
import type { PaymentInfo } from '@x402r/sdk'

// Set up viem clients
const publicClient = createPublicClient({
  chain: baseSepolia,
  transport: http(),
})

const account = privateKeyToAccount(process.env.PRIVATE_KEY as `0x${string}`)
const walletClient = createWalletClient({
  chain: baseSepolia,
  transport: http(),
  account,
})

// Create x402r client
const x402r = createX402r({
  publicClient,
  walletClient,
  operatorAddress: '0x...', // Replace with actual operator address
})

// Define payment info
const paymentInfo: PaymentInfo = {
  payer: account.address,
  receiver: '0x...', // Merchant address
  token: '0x0000000000000000000000000000000000000000', // Native ETH
  amount: BigInt('1000000000000000000'), // 1 ETH
  operator: '0x...', // Operator address
}

async function main() {
  // Get operator configuration
  const config = await x402r.operator.getConfig()
  console.log('Operator Config:', config)

  // Check payment state
  const state = await x402r.payment.getState(paymentInfo)
  console.log('Payment State:', {
    isAuthorized: state[0],
    authorizedAmount: state[1],
    chargedAmount: state[2],
  })

  // Get detailed amounts
  const amounts = await x402r.payment.getAmounts(paymentInfo)
  console.log('Payment Amounts:', amounts)
}

main().catch(console.error)
```
